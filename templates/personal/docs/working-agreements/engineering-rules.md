# 共通ルール

1. スキーマファーストを厳守する
   API スキーマ（GraphQL / OpenAPI 等）を先に定義・更新してから、バックエンド / フロントエンドを実装する。
2. 命名規則を統一する
   DB・API・バックエンド・フロントエンドで命名を揃え、レイヤー間の揺れをなくす。
3. 認証のコアロジックはバックエンド側に置く
   フロントエンドはセッション管理の補助のみを担当する。
4. デザインシステムはラッパー経由で使う
   UI ライブラリを直接利用せず、ラッパーコンポーネントを介して利用する。
5. 新機能にはテストを同時に追加する
   実装のみ先行させず、検証コードまでセットで仕上げる。
6. Linter / Formatter 設定に従う
   警告を放置せず、整形ルールの逸脱を残さない。
7. 機密情報は暗号化して保存する
   OAuth トークンや API キーなどのシークレットを平文保存しない。
